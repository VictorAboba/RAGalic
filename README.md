# RAGalic

Система Retrieval-Augmented Generation (RAG) для работы с PDF-документами с иерархической структурой и гибридным поиском.

## Возможности

- **Извлечение текста из PDF** — использует Docling с OCR (EasyOCR) для распознавания текста с отсканированных документов
- **Иерархическое индексирование** — автоматическое построение дерева документа с агрегацией метаданных на разных уровнях
- **Гибридный поиск** — комбинирует dense (семантический) и sparse (BM25) векторный поиск с использованием RRF-фьюжина
- **Beam Search** и **Branch Search** — алгоритмы поиска для навигации по иерархической структуре документа
- **LLM-метаданные** — генерация описаний и ключевых слов с помощью больших языковых моделей

## Установка

1. Клонируйте репозиторий и перейдите в директорию проекта:
```bash
cd RAGalic
```

2. Установите зависимости через uv:
```bash
uv sync
```

3. Скопируйте файл конфигурации:
```bash
cp .env.example .env
```

4. Настройте переменные окружения в `.env`:
```
API_KEY="ваш_api_ключ"
URL_BASE="https://api.proxyapi.ru/openrouter/v1"
MODEL="z-ai/glm-4.7-flash"
```

## Использование

### Индексация документа

Для индексирования PDF-документа используйте функцию `build_tree`:

```python
from pathlib import Path
from rag_lib.build_tree import build_tree

path = Path("path/to/your/document.pdf")
build_tree(path, width=3, batch_size=5)
```

Параметры:
- `width` — количество дочерних узлов, объединяемых в родительский (по умолчанию 3)
- `batch_size` — размер батча для записи в БД (по умолчанию 5)

### Поиск по документам

#### Branch Search

Иерархический поиск, начинающий с корневых узлов:

```python
from rag_lib.search import branch_search

results = branch_search(query="ваш_запрос", num_roots=3)
for chunk in results:
    print(f"{chunk.file_name} (страницы {chunk.page_start}-{chunk.page_end}):\n{chunk.text[:200]}...")
```

#### Beam Search

Расширенный поиск с итеративным сужением:

```python
from rag_lib.search import beam_search

results = beam_search(query="ваш_запрос", beam_width=3)
```

## Архитектура алгоритмов

### 1. Извлечение текста (Chunking)

Проект использует [Docling](https://ds4sd.github.io/docling/) — высокоточный конвертер документов с поддержкой:
- Извлечения текста из PDF
- OCR через EasyOCR для отсканированных документов
- Распознавания таблиц

Текст обрабатывается от переносов строк (`fix_hyphenation`), после чего разбивается постранично.

### 2. Построение дерева документа

Алгоритм `build_tree` создаёт иерархическую структуру:

1. **Листья (страницы)** — каждая страница документа становится листом, для неё LLM генерирует:
   - `description` — краткое техническое описание (3-5 предложений)
   - `keywords` — 5-10 ключевых слов для BM25-поиска

2. **Агрегация в родительские узлы** — каждые `width` узлов объединяются в родительский, LLM генерирует обобщающее описание на основе метаданных дочерних узлов

3. **Корневой узел** — вершина дерева с `parent_id = -1`, представляющая весь документ

### 3. Векторное индексирование

Используется [Qdrant](https://qdrant.tech/) с гибридным поиском:

- **Dense embeddings** — `jinaai/jina-embeddings-v3` для семантического поиска
- **Sparse embeddings** — BM25 для точного ключевого поиска

Векторы хранятся в отдельных пространствах (`dense` и `sparse`) и объединяются через **Reciprocal Rank Fusion (RRF)**:

```
RRF_score(doc) = Σ 1 / (k + rank(doc))
```

где `k = 60` (константа сглаживания).

### 4. Поисковые алгоритмы

#### Branch Search

```
1. Найти корневые узлы (top-k) гибридным поиском
2. Для каждого корня:
   a. Сравнить родителя с его детьми
   b. Если дети релевантнее — добавить их в следующую итерацию
   c. Если родитель лучше детей — это финальный результат
3. Повторять до тех пор, пока есть более релевантные дети или достигли листа.
```

#### Beam Search

```
1. Найти корневые узлы (beam_width)
2. Итеративно:
   a. Сравнить всех родителей с их детьми
   b. Оставить top-k результатов с учётом сужения (suppression)
3. Вернуть финальные узлы
```

Логика сужения:
- Если ребёнок лучше родителя → исключаем родителя
- Если родитель лучше детей → исключаем всех детей
